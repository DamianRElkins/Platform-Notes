# Company Data Requirements for Platform Upload

Basic Doc Requirements:
What data would be best for upload
Companies may not know what is actually usefull

## What should I provide?

Ideal Data Supplied

Standardized Code (ISIC, NACE, etc.) | Internal Category Name | Item Name | Item Categorisation Heirarchy (EX. Whitbread) | Supplier Name | Supplier Operations Location | Spend | Date

Minimal Data Supplied

Item Name | Supplier Name | Spend

## What should my data look like?

### Ideal Item Data Standard

Preface: Items will always be placed into the best fitting category based on available information, only

Item and Category names should be as granular and descriptive as possible.

You should supply us with as much of your organisation's item naming/categorisation as possible.

Example Categorisation of an order of ready-mix concrete:

Good Example: Construction | Materials | Concrete

Bad Example: Development

Why?

The first example allows us to assign the best fitting category to your expense because it is specific, and there can be no confusion about what it is referring to.

The second example makes assigning an appropriate category difficult. First, because Development could apply to multiple kinds of development (Construction Development, Software Development). Second, because it is not specific about what part of the development it is talking about (Architectural Consulting, Construction Labourers, Materials).

### Ideal Supplier Standard

Supplier names should not have variants unless it is desired for them to be calculated separately (EX. British Water and British Water PO Box. xxxxx)

The country the supplier is based in can be provided for improved accuracy.

The exact location the supplier operates in or materials are sourced from should be provided for deeper levels of analysis. (Coordinates or Addresses)
